# Extended Dispute Simulations (v2.1)

## Scenario 4: Jury Collusion Attempt
**Case**: 3 assistants try to collude to protect one of their own.
**Mitigation**: Random selection from large pool + quadratic voting makes collusion extremely difficult.
**Outcome**: Collusion fails. All 3 lose reputation.

## Scenario 5: Coordinated False Claims by Users
**Case**: Group of users files multiple false claims against the same assistant.
**Mitigation**: Reputation system + minimum stake to raise dispute + outcome voting flags suspicious patterns.
**Outcome**: All false claims rejected. Users lose bonds + reputation.

## Scenario 6: High-Value Dispute with Appeal
**Case**: $400 order with clear evidence on both sides.
**Process**: First jury rules for assistant. User appeals with 2x bond.
**Larger Jury (21 members)** rules for user.
**Outcome**: User wins + assistant loses 40% collateral. Appeal bond returned to user.

**Key Insight**: The system handles both simple and complex cases fairly while maintaining strong anti-gaming protections.
